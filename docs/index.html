SprayCraft: Graph-Based Route Optimization
for Variable Rate Precision Spraying

An A-CPS framework that models disease spread as spatial graphs, detects infection hotspots via message passing, and computes near-optimal drone spray routes using TSP formulation.

Kiran K. Kethineni, Saraju P. Mohanty, Elias Kougianos, Sanjukta Bhowmick, Laavanya Rachakonda
University of North Texas  ·  University of North Carolina Wilmington

The Problem

Precision Agriculture Needs Smarter Spraying

Plant diseases cause up to 20% of global produce loss. Yet conventional uniform pesticide spraying wastes resources, damages ecosystems, and promotes crop resistance — because disease is never uniform.

🗺️

Spatial Heterogeneity

Disease severity clusters spatially — hotspots near infection sources are far more dangerous than peripheral instances. Uniform treatment ignores this entirely.

💧

Dosage Must Vary

High-probability infection hotspots need more pesticide. Low-risk areas need less. No existing system computes this from spatial data automatically.

🚁

Drone Routing is Hard

A spraying drone must visit all diseased locations efficiently. This is the Traveling Salesman Problem — NP-hard and critical for reducing battery and time cost.

⚙️

Diverse Hardware

Agricultural drones range from fixed flow-rate sprayers to variable-rate (PWM/PID) systems. Solutions must adapt to both without hardware-specific redesign.

The Solution

A Unified Two-Stage Framework

SprayCraft unifies hotspot detection and route optimization in a single graph-theoretic pipeline — from raw disease location data to GPS-ready drone waypoints.

1

Graph Construction

Each diseased instance becomes a node. Edges connect nodes within pathogen dispersal radius, weighted by inverse distance.

2

Message Passing

Iterative neighbor aggregation updates each node's feature, surfacing hotspots as nodes with the highest aggregated infection area.

3

Hotspot Scoring

Normalized node features become hotspot probability scores — directly driving variable spray dosages.

4

TSP Route

Christofides algorithm computes a near-optimal Hamiltonian circuit visiting all diseased locations (1.5× approximation guarantee).

5

Coverage Paths

Boustrophedon serpentine paths at each location ensure 100% coverage. Spacing adapts to spray radius derived from hotspot score.

6

GPS Waypoints

Meter-based coordinates convert to GPS lat/long via Haversine formula, producing drone-ready flight plans.

Message Passing (Stage 1)

# Graph: G = (V, E, F) where F = node feature = diseased area # Edge weight W(u,v) = distance between nodes u and v h_u(k+1) = UPDATE( h_u(k), AGG( { h_v(k) · 1/W(u,v) : v ∈ N(u) } ) ) # After k iterations: higher h_u → higher hotspot probability # Normalize: p_u = h_u / max(h)

Sprayer Adaptation (Stage 2)

Variable Rate Sprayer (VRS_BuiltIn = True)

PWM/PID-controlled flow rate. Dosage is modulated directly by hotspot probability score.

spray_radius = 2 - 1 × hotspot_probability
flow_rate ∝ hotspot_probability
altitude = constant

Constant Rate Sprayer (VRS_BuiltIn = False)

Fixed flow rate. Effective area controlled by adjusting flight altitude (conical spray geometry).

Area = π × R²
R = f(altitude, cone_angle)
altitude = g(hotspot_probability)

Technical Highlights

Graph Theory

Spatial Graph Representation

Disease instances map to graph nodes with edges encoding pathogen dispersal proximity and weighted by inverse distance.

GNN

Graph Message Passing

Neighbors share infection signals iteratively. Multi-hop aggregation captures cluster effects that single-node analysis misses.

Optimization

Christofides TSP

MST + minimum-weight perfect matching on odd-degree nodes → Eulerian circuit → Hamiltonian circuit. Provably within 1.5× of optimal.

Coverage

Boustrophedon Paths

Parallel serpentine sweeps with spacing equal to 2× spray radius. Guarantees 100% area coverage with zero overlaps or gaps.

Hardware

Dual Sprayer Support

A single parameter switch (VRS_BuiltIn) enables the same pipeline to drive both variable-rate and constant-rate drone sprayers.

Navigation

GPS Coordinate Conversion

Haversine formula converts pixel/meter waypoints to GPS lat/long coordinates, making output directly consumable by commercial drones.

Variable Rate

Adaptive Dosage Allocation

Variable-rate mode concentrates pesticide in hotspot zones while reducing dosage at peripheral locations — cutting waste while protecting the most vulnerable areas.

Route Efficiency

Notebook Structure

Section 1

Data Loading

Load annotated apple leaf disease dataset. Parse bounding boxes and disease class labels (Black Rot, Cedar Apple Rust, Apple Scab).

Section 2

Graph Construction

Build spatial graph from disease instance coordinates. Set node features = diseased area. Connect nodes within dispersal radius threshold.

Section 3

Message Passing

Apply iterative neighbor aggregation. Visualize node feature evolution across iterations. Compute and normalize hotspot probability scores.

Section 4

Route Optimization

Compute MST and apply Christofides algorithm. Output near-optimal TSP tour with total distance and ordered node sequence.

Section 5

Path Generation

Generate boustrophedon coverage paths per location. Adapt spacing and altitude to hotspot scores. Render 3D flight altitude maps.

Section 6

GPS Output

Convert meter-based waypoints to GPS coordinates using Haversine formula. Produce drone-ready flight plan with lat/long waypoints.
